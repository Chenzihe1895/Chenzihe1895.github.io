  • 问题总结：
    • Java NIO 报java.nio.charset.MalformedInputException: Input length = 1异常
      • 这个是由于maven在compile默认使用GBK编码，而项目文件中的yml在解析时需要使用utf-8编码，所以启动springboot会报错。
    • maven
    • 判空问题：基本每个方法都要判null，时刻判null是个好的编程习惯(因为类型转换的时候，会出现空值问题，如path.toFile，如果对应路径上的文件不存在的话，会变成null)
  • 公司的spring项目和我以前接触到的区别很大，与数据库的交互比较少，主要还是对数据流的处理，因为FlowPipe项目主要是数据清洗、数据落盘，我尚不熟悉业务、数据落盘的工作由其他同事负责。
  • 新见到了一个snakeyaml，可以通过maven配置，能够将yaml文档转换为Java对象，以及将java对象序列化为java文档
    • yaml类的实现是线程非安全的，不同的线程必须有自己的yaml实例。
    • yaml是json的超集，可以书写json类型的maps和sequence。
